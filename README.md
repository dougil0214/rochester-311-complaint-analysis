Rochester 311 Complaint Analysis
Overview

This project analyzes Rochesterâ€™s 311 service request data to identify where complaints occur, what residents report most often, and how patterns vary by geography and time. It simulates a real municipal data analyst workflow from raw data to dashboard.

Objectives

Identify areas with the highest volume of 311 complaints

Determine the most common complaint types citywide and by area

Analyze complaint trends over time

Communicate findings through a clear, map-based dashboard

Data

Source: City of Rochester Open Data Portal

Dataset: 311 Service Requests

Key fields: complaint type, created date, geographic area, status

Raw data is preserved; analysis is performed on cleaned copies.

Tools

SQL (MySQL) for analysis

Power BI for visualization

GitHub for version control

Asana for project management
